# Задача 04 — Голосование зрителей

**Приоритет:** 🟠 P1 · **Статус:** ⏳ обдумываем (детальный план распишем когда подойдём)

## Цель простым языком

Чтобы зритель Рифмабеса мог зайти на сайт, нажать «Войти», авторизоваться **без боли** (без длинных регистраций и пароля для очередного сайта), и ставить **свои** оценки трекам и выбирать **своего** победителя в раундах баттлов.

Принцип: **один аккаунт = один голос на сущность**. Можно поменять, но не суммируется.

## Голосование по типам контента

- **Альбом / трек** — оценка по 10-балльной шкале (тот же ползунок что у Рифмабеса).
- **Standard баттл (1v1 или team)** — для каждого раунда «победил A / B / ничья», плюс итоговый победитель.
- **Дезматч-баттл** (см. отдельная задача в [BACKLOG.md](../../docs/BACKLOG.md)) — выбор одного победителя из N участников. НЕ «A/B/ничья» — модель другая.

## Открытые вопросы

1. **Какой провайдер логина?** Google / VK / magic-link на email / несколько сразу.
2. **Что считать голосом для баттла?** Только итог или каждый раунд тоже.
3. **Защита от накрутки.** Достаточно ли «1 аккаунт = 1 голос», или ещё подтверждение почты, ограничение свежих акк-аунтов, captcha при первом голосе.
4. **Можно ли голосовать без логина?** Скорее нет — спам неминуем.

- Параллельно с задачей 05 (UI «две оценки»).
- Для дезматчей — после задачи «Дезматч-баттлы» из [BACKLOG.md](../../docs/BACKLOG.md).

   - **Превью / обложка** — превью ролика в максимальном разрешении.
   - **Ссылка на оригинал** — на YouTube, как `MediaLink kind=original`.
   - **Дата релиза** — год публикации видео (нужен YouTube Data API, oEmbed не даёт).
   - **Стороны / участники** — эвристически из названия.
3. **Раунды и победители** заполняются **вручную самим Рифмабесом** — это его субъективная оценка, импорт не угадает.

## Форматы баттлов которые надо различать

Из реального исследования 8 роликов 2026-05-18 (см. анализ ниже) — баттлы бывают разные:

1. **Standard 1 vs 1.** `БИГА ONE vs ВИШНЯ` — два бойца, две стороны, раунды.
2. **Team battle.** `R1FMABES & VERCH.FATE vs ШУММ & SECTOR` — две команды по N человек, две стороны.
3. **Deathmatch.** `T!MMI x Hidji 17 x AVENUE SS х HPPD х АГЕНТ ДАН` — N участников **все против всех**, побеждает один из N. Канал «БОЛЬШЕ ЧЕМ БАТТЛ» делает именно такой формат.

**Текущая модель данных** в [src/main.tsx](../../src/main.tsx) (`BattleMetadata`) поддерживает только формат 1-2 (две стороны A/B). Формат «дезматч» **не покрыт** — нужно расширить модель (см. отдельная задача в [BACKLOG.md](../../docs/BACKLOG.md) → «Дезматч-баттлы»). Без этой задачи дезматчи импортировать нельзя — некуда писать список из N участников.

## Эвристика парсинга названий

Анализ 8 роликов показал стабильные паттерны. Псевдокод парсера:

```ts
function parseBattleTitle(title: string, authorName: string) {
  // 1. Чистим хвост: хештеги, год в конце "| YYYY"
  let t = title.replace(/\s*#[\wА-яё]+/g, '').replace(/\s*\|\s*\d{4}\s*$/, '').trim();

  // 2. Разделяем "бойцы | турнир" — приоритетный разделитель |, потом " - "
  let fighters = t, tail = '';
  for (const sep of [' | ', ' - ', ' — ']) {
    const i = t.indexOf(sep);
    if (i > 0) { fighters = t.slice(0, i); tail = t.slice(i + sep.length); break; }
  }

  // 3. Versus-стиль: "A VS B (Tournament) Stage"
  const paren = fighters.match(/^(.+?)\s*\(([^)]+)\)\s*(.*)$/);
  let tournament = '', stage = '';
  if (paren) { fighters = paren[1]; tournament = paren[2]; stage = paren[3]; }

  // 4. Турнир + стадия из хвоста: "КУБОК МЦ: XX" -> tournament=КУБОК МЦ, stage=XX
  if (!tournament && tail) {
    const m = tail.match(/^([^:]+?)(?::\s*(.+))?$/);
    tournament = m?.[1].trim() ?? tail;
    stage = m?.[2]?.trim() ?? stage;
  }

  // 5. Определяем формат:
  // - Если есть " vs ", " VS ", " против " — это 1v1 или team (standard)
  // - Иначе если канал = "БОЛЬШЕ ЧЕМ БАТТЛ" ИЛИ если в fighters >2 имён через x/х — deathmatch
  const vsMatch = fighters.match(/^(.+?)\s+(?:vs|VS|Vs|против)\s+(.+)$/);
  const split = (s: string) =>
    s.split(/\s+(?:x|х|&|\+)\s+/i)
     .map(p => p.replace(/\s+aka\s+.+$/i, '').trim())  // схлопываем "RAM aka РАМИРЕС" в одного
     .filter(Boolean);

  if (vsMatch) {
    return {
      format: 'standard' as const,
      sideA: split(vsMatch[1]),
      sideB: split(vsMatch[2]),
      tournament,
      stage,
    };
  }
  // Дезматч: все участники без сторон
  if (/БОЛЬШЕ ЧЕМ БАТТЛ|DEATHMATCH/i.test(authorName + ' ' + title)) {
    return {
      format: 'deathmatch' as const,
      participants: split(fighters),
      tournament: tournament || authorName,
      stage,
    };
  }
  // Не разобрали — отдаём весь fighters одной строкой, Рифмабес доразберёт
  return { format: 'unknown' as const, raw: fighters, tournament, stage };
}
```

## Поддерживаемые источники (в первую очередь)

- **КУБОК МЦ** (`youtube.com/@kbkmc`) — формат `A vs B | КУБОК МЦ: <сезон>`. 5 из 6 роликов парсятся идеально.
- **Versus Battle** — формат `A VS B (Versus: Fresh Blood N) Этап M | YYYY`. Парсится, нужна доп. регулярка на хвостовой год.
- **БОЛЬШЕ ЧЕМ БАТТЛ** (DEATHMATCH SEASON) — N участников через `x` / `х`, формат дезматча.
- **Tiger Battle**, **140 BPM Battle**, etc. — на будущее, разберёмся когда понадобятся.

## Технический план

- **Подход 1 (oEmbed, без API key):** `https://www.youtube.com/oembed?url=...&format=json`. Бесплатно, без регистрации. Даёт: `title`, `author_name`, `author_url`, `thumbnail_url`. **Год публикации НЕ даёт.** Работает из браузера без CORS-проблем.
- **Подход 2 (YouTube Data API v3):** добавляет дату публикации, описание, длительность. Нужен API key (бесплатный, 10k quota/день, регистрация в Google Cloud Console). Лимит на запрос ~3-5 quota — на наши объёмы более чем достаточно.

Рекомендация: **сначала делаем oEmbed-вариант** (быстро, бесплатно, никаких ключей), потом если будем хотеть автоподстановку года — добавляем Data API.

При выборе подхода 2: API key кладём в `.env` нашего VPS-прокси ([server/yandex-proxy/](../../server/yandex-proxy/)), добавляем эндпоинт `/youtube/import?url=...`. Прокси и так уже работает — просто новый роут.

- `N'rage`, `T!MMI` — апостроф и `!` в нике, не разделитель.
- `СЫНОЧКА-КОРЗИНОЧКА` — дефис **внутри** имени, не разделитель.
