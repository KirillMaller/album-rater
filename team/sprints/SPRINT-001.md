# Задача 04 — Голосование зрителей

**Приоритет:** 🟠 P1 · **Статус:** ⏳ обдумываем (детальный план распишем когда подойдём)

## Цель простым языком

Чтобы зритель Рифмабеса мог зайти на сайт, нажать «Войти», авторизоваться **без боли** (без длинных регистраций и пароля для очередного сайта), и ставить **свои** оценки трекам и выбирать **своего** победителя в раундах баттлов.

Принцип: **один аккаунт = один голос на сущность**. Можно поменять, но не суммируется.

## Открытые вопросы

1. **Какой провайдер логина?** Google / VK / magic-link на email / несколько сразу.
2. **Что считать голосом для баттла?** Только итог или каждый раунд тоже.
3. **Защита от накрутки.** Достаточно ли «1 аккаунт = 1 голос», или ещё подтверждение почты, ограничение свежих акк-аунтов, captcha при первом голосе.
4. **Можно ли голосовать без логина?** Скорее нет — спам неминуем.

# Задача 08 — Импорт баттлов с YouTube

**Приоритет:** 🟠 P1 · **Статус:** ⏳ обдумываем (детальный план распишем когда подойдём)

## Цель простым языком

Чтобы Рифмабесу было откуда импортировать баттлы. У альбомов и треков есть Яндекс.Музыка — там всё уже работает. У баттлов источник = **YouTube-каналы баттл-проектов** (Кубок МЦ, Versus, и т.д.). Хочется: вставил ссылку с YouTube → автоматически подтянулись название, кто против кого, превью, ссылка на видео.

## Что увидит пользователь

1. В редакторе при типе «баттл» в блоке импорта поле принимает ссылку YouTube.
2. Вставил ссылку → нажал «Импортировать» → заполнилось:
   - **Название** видео (из метаданных YouTube): «Кубок МЦ. 5 сезон. 1/8: Иван vs Пётр» и т.п.
   - **Превью / обложка** — превью ролика в максимальном разрешении.
   - **Ссылка на оригинал** — на YouTube, как `MediaLink kind=original`.
   - **Дата релиза** — год публикации видео.
3. **Стороны A и B** парсятся **эвристически из названия** (ищем разделители `vs`, `против`, `—`, `vs.`). Если не нашли — оставляем пустыми, Рифмабес дозаполняет руками.
4. **Раунды и победители** заполняются **вручную самим Рифмабесом** — это его субъективная оценка, импорт не угадает.

## Технический план (черновик)

- **YouTube Data API v3** — публичный, бесплатный лимит (10 тыс. quota units/день, импорт одного видео ~3-5 units). Доступен глобально, **геоблоков нет**.
- Нужен **API key** (зарегать в Google Cloud Console, бесплатно).
- Реализация: добавить эндпоинт `/youtube/import?url=...` в наш существующий VPS-прокси ([server/yandex-proxy/](../../server/yandex-proxy/) — можно переименовать в `media-proxy`). API key хранится в `.env` на сервере, в браузер не утекает.
- Эвристика парсинга названия: регулярка на `(?:vs|против|—|×)` между двумя именами.

## Поддерживаемые источники (в первую очередь)

- **Кубок МЦ** (`youtube.com/@kubokmts`) — конкретно его упоминал Кирилл.
- **Versus Battle**.
- **140 BPM Battle**.
- Любой YouTube-канал, в принципе — формат универсальный.

## Открытые вопросы

- **Что делать с многораундовыми баттлами в одном видео.** Раунды-то всё равно ставит руками Рифмабес — импорт даёт только верхушку (название, превью, ссылку).
- **Как достать имена бойцов из тех видео где формат названия странный.** Возможно потребуется ручной список регулярок под каждый канал.
- **Лимиты YouTube API** — 10k quota в день более чем хватит, но если когда-нибудь будем импортировать пачкой — стоит закешировать.

## Зависимости

- Никаких — можно начинать когда захотим.
- Хорошо бы заодно с задачей 07 (другие источники), чтобы прокси превратить в единый «media import gateway».
