# Улучшения маршрута: аукцион → колесо → победитель

Аудит всего пути от создания аукциона до победителя — что упростить, починить, убрать (дубли),
что добавить для удобства **админа (Рифмабеса)** и **зрителей**. Составлено 2026-07-20 на основе:
(1) разбора кода `src/main.tsx`, (2) веб-исследования похожих сервисов (wheelofnames, pickerwheel,
DonationAlerts, StreamElements, TipeeeStream, Streamloots, Twitch predictions), (3) практик навигации/IA.
_Codex как третье мнение подключить не удалось — устарел CLI (0.130.0), несовместим с моделью; можно
обновить и переспросить позже._

Приоритеты: 🔴 P0 (важно/риск на стриме) · 🟠 P1 · 🟡 P2 · 🟢 P3. Сложность: low / med / high.
Пункты, уже заведённые в [BACKLOG.md](BACKLOG.md), помечены `[BL#N]` — не дублировать.

## Как устроено сейчас (кратко)
- **Публичное:** `/auctions` (очередь), `/auctions/wheel` (живой зрительский экран, опрос 2.5с).
- **Админ:** `/admin/auctions` — управление позициями + кнопка «Колесо» разворачивает панель.
- **Поток:** добавить позицию → накидать суммы донатов (кнопка «+») → открыть колесо → выбрать
  категории и эмодзи → «Зафиксировать участников» → «Крутить» много раз (каждый спин выбивает
  одного) → победитель → результат висит 24ч (или крестик убирает сразу).

## B. Админ (Рифмабес) — удобство и риски на живом стриме
- 🔴 **B1. Один активный розыгрыш за раз + авто-возврат к нему** `[BL#3]`. Сейчас сессия ищется по
  точному совпадению категорий → можно наплодить дубли и не найти своё колесо. Заходишь в
  админку — кидает сразу на текущий розыгрыш, пока не отменил/не доиграл. **med.** (Главная путаница.)
- 🔴 **B2. Undo последнего спина.** Каждый спин необратимо выбивает участника на сервере; единственный
  откат — «Отменить сессию» (теряется весь розыгрыш). Один промах на стриме = начинать заново.
  Нужна кнопка «отменить последний раунд»: удалить последний `wheel_round` и вернуть участника в
  игру. Требует серверного RPC. **high — но это самый большой риск на живом эфире.**
- 🟠 **B3. Один селектор категорий.** У панели колеса свои «пилюли» категорий, а над ней — вкладки
  категорий страницы; они не синхронизированы (панель берёт категорию один раз при открытии). Два
  разных выбора на одном экране путают. Оставить один источник. **med.**
- 🟡 **B4. Автообновление панели админа во время `locked`.** Публичная страница опрашивает базу
  каждые 2.5с, а админская — только после своего спина, и при рассинхроне просит нажать «Обновить».
  Лёгкий поллинг убрал бы тупиковые состояния. **low/med.**
- 🟡 **B5. Подпись колонки «%» в подготовке.** В подготовке «%» = доля банка (больше денег —
  больше %), а после старта та же колонка = «% вылет» (больше денег — меньше %). Смысл незаметно
  переворачивается. Подписать столбец в подготовке («доля банка») или показывать шанс вылета сразу. **low.**
- 🟡 **B6. «Начать разбор из результата».** Когда колесо определило победителя — кнопка, чтобы сразу
  проставить позиции `reviewed_at` / отправить в каталог, не вбивая руками. Замыкает цикл
  колесо → каталог. **low.**
- 🟢 **B7. Ссылка «открыть зрительское колесо ↗» из админ-панели.** Спин только в админке, живой
  экран — отдельный URL; явная ссылка, чтобы не держать две вкладки наугад. **low.**
- 🟢 **B8. Предпросмотр/«сухой» спин (только локально, без трансляции зрителям).** Стример видит
  колесо и шансы до того, как пойдёт в эфир. **med.**
- 🟢 **B9. Настройки: звук вкл/выкл, длительность спина, reduced-motion** (часть уже есть). **low.**

## C. Зрители — понятность и вовлечение
- 🟠 **C1. Замедление + тик-звук колеса.** Главный апгрейд ощущения по всем источникам: плавное
  торможение + «тик-тик-тик» к остановке делают спин «выстраданным». Сейчас звука нет (ждёт файла
  от Рифмабеса, Этап 4). Добавить easing + тик, кнопку «выкл звук». **med.** (частично запланировано.)
- 🟡 **C2. Отдельный «бит» на каждом выбывании.** У тебя инверсия: драма не в победе, а в «X ВЫЛЕТАЕТ».
  Дать заметную вспышку «❌ выбывает: X» на каждом спине, а конфетти оставить только финальному
  победителю (конфетти на финале уже есть; блок «Выбывает» тоже есть — усилить). **low/med.**
- 🟡 **C3. Список «выбыли по порядку» на зрительском экране.** Кто зашёл поздно — видит историю
  (1-й вылетел, 2-й…). Данные уже есть («История раундов») — показать нагляднее. **low.**
- 🟡 **C4. Кнопки быстрой суммы должны СКЛАДЫВАТЬСЯ, а не заменять.** В `AddAmountModal` «+100»
  потом «+300» ставит 300, а не 400 — хотя подпись «+» намекает на накопление. **low.**
- 🟡 **C5. Прогресс-бар у позиции — «сколько до обгона верхней».** Донат-цели у всех сервисов
  показывают заполняющуюся полоску: абстрактное число → полоска, которую хочется добить. **med.**
- 🟡 **C6. Признание топ-донатеров у позиции** (кто больше всех закинул). Именное признание —
  проверенный драйвер донатов. Требует хранить личность донатера + приватность/согласие (как уже
  есть consent-флоу). **med.** (обдумать приватность.)
- 🟢 **C7. Однострочное пояснение механики и на экране победителя** (сейчас баннер только на экране
  вращения; кто зашёл на кубок — без контекста). **low.**
- 🟢 **C8. Анимация «обгона» при донате** — когда донат поднимает позицию выше соседа, анимировать
  перестановку + вспышка «обогнал!». Это и есть драма очереди. **med.**
- 🟢 **C9. Шэр-карточка победителя** (обложка + «следующий разбор, выбрано колесом …») для чата/соцсетей. **med.**
