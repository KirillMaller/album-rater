### Онлайн-баттл — отдельный тип записи

Кирилл прямо обозначил (2026-05-18) что онлайн-баттл — это **особый формат**, у него **другие правила оценки** и должна быть **другая карточка**, не как у обычного баттла с раундами. Поэтому делаем как отдельный `ItemType`, не как подформат.

**Что такое:** соревнование без живого выступления — каждая сторона выкладывает свой материал (треки-диссы) в соцсетях. Структура: «два мини-альбома один против другого».

**Что нужно:**

- В `ItemType` добавить значение `'online_battle'`.
- Миграция БД: расширить check-constraint `rated_items.type` (это будет миграция `003_add_movie_anime_online_types.sql` если идём с фильмом/аниме одним пакетом, иначе `003_add_online_battle.sql`).
- Метаданные: стороны A и B, плюс **два списка треков** (как у альбома, но раздельно для каждой стороны). Раундов нет — оценка по каждому треку + общая.
- В редакторе при выборе типа «Онлайн-баттл» — форма с двумя секциями «Сторона A: треки» / «Сторона B: треки» и общим победителем.
- На странице записи — рендер «двух треклистов рядом» с оценками. Победитель — сводно.
- В каталоге — отдельная вкладка / фильтр для онлайн-баттлов.

**Открытые вопросы:**
- Какой источник для импорта? YouTube (один общий ролик), отдельные ссылки на YT-видео каждой стороны, или ссылки на треки в Яндекс.Музыке?
- Кто выигрывает — выбор R1Fmabes, или по сумме оценок треков?

### Дезматч-баттлы (N участников все против всех)

Сейчас модель `BattleMetadata` в [src/main.tsx](../src/main.tsx) поддерживает только классический баттл из двух сторон A/B. Но в реальности есть **третий формат** — **дезматч**: N участников, все против всех, побеждает один. Канал «БОЛЬШЕ ЧЕМ БАТТЛ» (DEATHMATCH SEASON) делает именно такой формат.

**Пример:** `БОЛЬШЕ ЧЕМ БАТТЛ | T!MMI x Hidji 17 x AVENUE SS х HPPD х АГЕНТ ДАН` — 5 участников, все против всех.

**Что нужно:**

- В тип `BattleMetadata` добавить поле `format: 'standard' | 'deathmatch'` (default `'standard'`).
- Для `standard` — текущая модель (sideA, sideB, rounds, finalWinner).
- Для `deathmatch` — новые поля: `participants: BattleParticipant[]` и `winner: string` (id одного из участников). Раунды для дезматча — опциональны, обычно дезматч это один заход.
- В редакторе [src/main.tsx](../src/main.tsx) — переключатель формата при создании баттла. Для дезматча — список участников (можно добавлять/удалять), выбор победителя.
- В голосовании зрителей (см. отдельная P1) — для дезматча голос = «кто из N победил», не «A/B/ничья».

**Миграция БД:** ничего не нужно — `metadata` это `jsonb`, новое поле живёт там же.

### Импорт по ссылке для других платформ

Яндекс уже работает (через VPS-прокси). Расширить идею на другие источники:

- **Apple Music / iTunes** — Search API, без авторизации, обложки качественные. **Самый простой кандидат**, не требует VPS-прокси (геоблока нет).
- **Spotify** — Web API, нужен client_id/secret, OAuth-данс. Метаданные отличные.
- **Bandcamp** — нет публичного API, парсить HTML.
- **Kinopoisk / IMDb** для фильмов, **Shikimori / MyAnimeList** для аниме.

Логика: фронт определяет источник по `new URL(input).hostname`, фронт зовёт соответствующий импортёр. Яндекс — через VPS, остальные могут быть прямо в браузере (если CORS позволяет) или через тот же VPS как унифицированный шлюз.
