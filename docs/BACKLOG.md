### Голосование зрителей (лёгкий логин + одна оценка с акка)

**Цель:** зрители могут ставить свою оценку трекам и выбирать победителя в раундах баттлов. С одного аккаунта — один голос на сущность.

**Требования:**

- Лёгкий способ зайти. Кандидаты: Google OAuth через Supabase Auth, VK OAuth, magic-link на email без пароля. Главное — **не пароль**, чтобы зритель не отваливался на регистрации.
- Один аккаунт = один голос на трек / на раунд / на итог баттла. Голос можно поменять, но не суммируется.
- Защита от накрутки на уровне аккаунта.
- В БД: новая таблица `viewer_votes` (item_id, track_id или round_id, viewer_id, score / winner_side), с уникальным индексом.

**Открытые вопросы:** какой провайдер логина выбрать; нужны ли ограничения по возрасту/региону; что считать «голосом» для баттла — только итог, или каждый раунд тоже.

### Две независимые оценки на карточке: стример vs зрители

После голосования (см. выше) — на странице записи показывать **две оценки рядом**:

- Слева — оценка Рифмабеса (`final_score`).
- Справа — средняя оценка зрителей (из `viewer_votes`).

Для баттлов аналогично: «победитель по Рифмабесу» vs «победитель по большинству зрителей».

### Новые типы контента: фильм и аниме

Рифмабес смотрит фильмы и аниме по донатам зрителей. Сейчас типы записи — только альбом / баттл / трек.

- Поля специфичные хранить в `metadata jsonb`.

- Импорт по ссылке: Kinopoisk / IMDb / Shikimori / MyAnimeList — отдельная задача ниже.

### Импорт по ссылке для других платформ

Яндекс уже работает (через VPS-прокси). Расширить идею на другие источники:

- **Apple Music / iTunes** — Search API, без авторизации, обложки качественные. **Самый простой кандидат**, не требует VPS-прокси (геоблока нет).
- **Spotify** — Web API, нужен client_id/secret, OAuth-данс. Метаданные отличные.
- **Bandcamp** — нет публичного API, парсить HTML.
- **Kinopoisk / IMDb** для фильмов, **Shikimori / MyAnimeList** для аниме.

Логика: фронт определяет источник по `new URL(input).hostname`, фронт зовёт соответствующий импортёр. Яндекс — через VPS, остальные могут быть прямо в браузере (если CORS позволяет) или через тот же VPS как унифицированный шлюз.

### Импорт баттлов с YouTube

Сейчас у баттлов нет источника для автоимпорта — Рифмабес заводит их руками. У него есть конкретные YouTube-каналы (Кубок МЦ и другие), откуда хорошо бы тянуть метаданные ролика автоматически.

**Что импортируем:**
- Название видео → попытка распарсить «кто vs кто» эвристически.
- Превью (обложка) ролика.
- Год публикации.
- Ссылка на оригинал.

**Чего НЕ импортируем (это оценочное, ставит Рифмабес):**
- Раунды.
- Победители раундов.
- Итоговый победитель.

**Как:** YouTube Data API v3, бесплатно, без геоблоков. Добавить эндпоинт `/youtube/import` в наш VPS-прокси. См. задачу 08 в [текущем спринте](../team/sprints/SPRINT-001.md).
