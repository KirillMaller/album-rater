# ARCHITECTURE — R1fрейтинг

Описание продукта и устройства приложения. Простым языком — для продакта.

## Для кого это

**Рифмабес** — стример на YouTube / Twitch / Boosty. Чем занят:

- Баттл-рэпер (сам выступает + смотрит чужие баттлы, делает реакции).
- Реактор: слушает альбомы и треки, разбирает их на стримах.
- Смотрит фильмы и аниме «по донатам» от зрителей.
- Пишет свои треки.

Сайт нужен трём аудиториям:

1. **Рифмабес** — быстро заводит карточку «что слушал / смотрел / читал», ставит оценку, делится зрителям одной ссылкой вместо «найди в стриме на 2:14:30».
2. **Зрители** — заходят и видят всё в одном месте: что Рифмабес послушал, как оценил, на какие баттлы делал реакцию.
3. **Кирилл (админ)** — пока сам заводит и правит контент от имени Рифмабеса. Когда продукт устаканится, передаст ввод самому Рифмабесу.

## Что делает сайт сейчас

Сайт-каталог авторских оценок. Три типа записей:

- **Альбом** — список треков с оценкой каждого, итог считается как среднее (`scoreMode: 'auto'`). Можно вручную переопределить.
- **Баттл** — стороны A и B, любое количество раундов, у каждого раунда свой победитель (A / B / ничья), плюс итоговый победитель.
- **Отдельный трек** — одна ручная оценка (`scoreMode: 'manual'`), без треклиста.

К любой записи можно прицепить:

- Обложку, год, жанр, описание, ревью в Markdown.
- Ссылки на оригинал (Яндекс.Музыка, Spotify, Apple Music, YouTube Music, VK, SoundCloud, Bandcamp).
- Ссылки на реакции — **это видео самого Рифмабеса**, где он обозревает запись (YouTube, Boosty, Twitch, VK Видео, Rutube).

## Куда продукт движется (видение)

В порядке приоритета:

1. **Удобный ввод одной ссылкой.** Вставил URL из Яндекс.Музыки → подтянулись название, артист, год, обложка, треклист. Та же история для Spotify, Apple Music, Bandcamp в будущем. Цель — Рифмабес не должен набивать вручную.
2. **Голосование зрителей.** Лёгкий логин (не пароль на каждом сайте — что-то типа Google/VK OAuth или magic-link на почту). Один аккаунт = один голос на трек / на раунд баттла / на победителя баттла. Защита от накрутки на уровне аккаунта.
3. **Две оценки на карточке.** Слева оценка Рифмабеса, справа средняя оценка зрителей. Считаются независимо. То же самое для баттлов: «кого считает победителем Рифмабес» и «кого считают зрители большинством».
4. **Расширение типов контента.** Сейчас есть альбом / баттл / трек. В планах добавить:
   - **Фильм** — название, год, режиссёр, обложка, оценка, ссылка на источник по которому смотрели.
   - **Аниме** — то же + кол-во серий / сезон.
5. **Импорт баттлов с YouTube.** Для альбомов и треков источник — Яндекс.Музыка (уже работает). Для баттлов источник — YouTube-каналы баттл-проектов (Кубок МЦ, Versus, 140 BPM). Из ролика автоматически тянем название, превью, год, ссылку. Эвристически парсим «кто vs кто» из названия. Раунды и победители Рифмабес ставит руками — это его оценка.

## Пользователи

- **Анонимный посетитель** — сейчас видит только опубликованные записи каталога. В будущем сможет зарегистрироваться и голосовать.
- **Зритель с аккаунтом** (будущее) — авторизованный пользователь. Может ставить оценку трекам и выбирать победителя в раундах баттлов. Один голос на сущность с одного аккаунта.
- **Рифмабес** (будущее) — заведёт свой админский аккаунт, будет заполнять каталог сам. Сейчас его роль выполняет Кирилл.
- **Админ** (Кирилл) — логинится через Supabase Auth, заводит и редактирует записи. Чтобы получить права, `user_id` должен лежать в таблице `admin_users` (см. [db/migrations/001_init.sql:45-49](../db/migrations/001_init.sql#L45-L49)).

## Маршруты

Источник правды — `BrowserRouter` в [src/main.tsx](../src/main.tsx).

| Путь | Что показывает |
|---|---|
| `/` | Главная — каталог опубликованных записей. |
| `/item/:slug` | Страница записи: обложка, итоговая оценка, треклист или раунды баттла, ссылки, ревью. |
| `/admin` | Логин админа. |
| `/admin/items` | Список всех записей (включая черновики), кнопка «Добавить». |
| `/admin/items/new`, `/admin/items/:id` | Редактор записи. |

Деплой на GitHub Pages в подпуть `/album-rater/` — поэтому Vite собирается с `base: '/album-rater/'`, а [public/404.html](../public/404.html) разворачивает прямые ссылки.

## Хранение данных

Два режима, выбираются автоматически по наличию env-переменных:

### Демо-режим (без `.env.local`)

Всё в `localStorage` под ключом `r1frating-items-v1` ([src/main.tsx:124](../src/main.tsx#L124)). Авторизация — фейковая, флаг `r1frating-demo-admin`. Удобно для разработки без интернета.

### Supabase-режим (есть `.env.local`)

`VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` → `createClient` ([src/main.tsx:126-129](../src/main.tsx#L126-L129)). Авторизация через Supabase Auth (email + password). RLS отделяет публичные чтения от админских.

## Схема БД

Полный SQL — [db/migrations/001_init.sql](../db/migrations/001_init.sql). Кратко:

- **`rated_items`** — основная таблица записей. `type`, `slug` (уникальный), `final_score`, `score_mode`, `published`, `metadata jsonb` (для баттлов и Яндекс-источника).
- **`track_scores`** — треки с оценкой, привязаны к `rated_items` через `item_id`. Уникальная позиция в рамках одного item.
- **`media_links`** — ссылки на оригиналы и реакции. `kind in ('original', 'reaction')`.
- **`admin_users`** — кто админ. `user_id` ссылается на `auth.users`.

### RLS правила

- Все могут читать опубликованные `rated_items` и их треки + ссылки.
- Писать что угодно — только админ (`is_admin()` функция).
- Список админов виден только админам.

### Миграция 002

[db/migrations/002_add_track_type.sql](../db/migrations/002_add_track_type.sql) добавляет `'track'` в check-constraint и колонку `metadata jsonb`. **На живой Supabase ещё не накачена** — пока не накатим, сохранение треков и баттлов с раундами в проде не работает (демо-режим работает).

## Доменные типы (frontend)

Источник — [src/main.tsx:39-122](../src/main.tsx#L39-L122). Главное:

- `ItemType = 'album' | 'battle' | 'track'`
- `ScoreMode = 'auto' | 'manual'` — авто = среднее по трекам, manual = руками.
- `BattleSideKey = 'a' | 'b' | 'draw'` — победитель раунда или баттла.
- `ItemMetadata` — JSON-поле для произвольных данных: настройки баттла (`battle.sideA/sideB/rounds/finalWinner`) и пометка про Яндекс-источник (`yandex.albumId/trackId/sourceUrl`).
- `MediaLink.kind = 'original' | 'reaction'` — оригинал или реакция.

## Импорт из Яндекс.Музыки

Идея: вставляешь ссылку (альбом или конкретный трек), система сама подтягивает название, артиста, год, обложку, треклист.

### Как устроено сейчас

```
Браузер пользователя (GitHub Pages, HTTPS)
  ↓ кнопка «Импортировать» в редакторе
fetch(`${VITE_YANDEX_PROXY_URL}/yandex-music/import?url=...`)
  ↓ HTTPS-запрос на VPS
Caddy на 195.208.3.209 (HTTPS-фронт с Let's Encrypt)
  ↓ reverse_proxy на 127.0.0.1:3001
Node.js yandex-proxy (~110 строк, server/yandex-proxy/index.mjs)
  ↓ HTTP-запрос на api.music.yandex.net/albums/{id}/with-tracks
Яндекс.Музыка JSON API (видит русский IP — отдаёт 200 OK)
```

В ответе приходит готовый `YandexImportResult` с полями: `albumId`, `trackId`, `title`, `artist`, `year`, `genre`, `coverUrl`, `tracks[]`, `sourceUrl`. Дальше фронт ([src/main.tsx:879-925](../src/main.tsx#L879-L925)) заполняет форму редактора, ставит обложку, выбирает тип (`track` если 1 трек, `album` если больше).

### Почему именно так

Яндекс.Музыка **геоблокирует** свой API для не-российских IP — `api.music.yandex.net` отдаёт HTTP 451 любым серверам в EU/US. Из-за этого не работают:

- Supabase Edge Functions (Ireland) — пробовали 2026-05-18, провалились.
- Vercel / Cloudflare Workers — та же история.
- Прямой запрос из браузера — блокируется CORS (Яндекс не разрешает кросс-доменные).

Решение: маленький Node.js-прокси на VPS **в РФ** (тот же сервер где живёт Telegram-бот Кирилла `bot-napominalka`). Подробности по поддержке — в [server/yandex-proxy/README.md](../server/yandex-proxy/README.md).

### Локальная разработка

В `.env.local` указан `VITE_YANDEX_PROXY_URL=https://195.208.3.209.sslip.io` — даже в `npm run dev` импорт идёт на тот же VPS-прокси что и прод. Никаких отдельных dev-плагинов больше нет — `vite.config.ts` минимальный.

## Что важно помнить (gotchas)

- **`base: '/album-rater/'`** — все пути на проде с префиксом. Если что-то ломается с 404 — смотрим SPA-фоллбек.
- **Демо vs Supabase** определяется автоматически по env. Локально без `.env.local` будешь работать на `localStorage` и не заметишь.
- **`src/main.tsx` — монолит на ~1300 строк.** Не торопиться разбивать на компоненты, пока продукт ещё гуляет.
- **Импорт Яндекса завязан на работу VPS `bot-napominalka`.** Если он упадёт — кнопка «Импортировать» перестанет работать. См. [server/yandex-proxy/README.md](../server/yandex-proxy/README.md) → секция «Что делать, если перестанет работать».
- **Старая версия** [index.legacy-backup.html](../index.legacy-backup.html) — не трогать, это архивный калькулятор без БД.
