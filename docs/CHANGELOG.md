# CHANGELOG — R1fрейтинг

Что изменилось для пользователя. Без технического жаргона.

Формат: новые записи сверху, дата в формате `YYYY-MM-DD`.

---

## 2026-05-18

- **Аукционы — отдельный раздел сайта.** В шапке появились две новые ссылки: **«Аукционы»** и **«Правила»**. Аукционы — это очередь записей, на которые зрители скидывают донаты: чем больше собрано на запись — тем раньше Рифмабес её разбирает. Шесть категорий: альбомы, сериалы, фильмы, аниме, игры, баттлы. Переключаются вкладками, чтобы не листать длинный список. По умолчанию открываются «Альбомы». В очередь сразу залиты 41 запись со старых табличек: 26 альбомов, 10 аниме, 5 сериалов. Категории «фильмы», «игры» и «баттлы» пока пустые. Несколько записей (Киберпанк, Необъятный океан) помечены «Категория под вопросом — уточнить», поправь через админку. **Админ видит дополнительно**: кнопку «+ Добавить» над таблицей и иконки «Изменить» / «Удалить» у каждой строки прямо на публичной странице — не нужно прыгать в админку. Также есть отдельная страница `/admin/auctions` с тем же функционалом, плюс редактор правил.
- **Подсветка «зависших» аукционов.** Если запись с маленькой суммой (≤ 500 ₽) долго не обновляется (никто не докидывает донаты) — она автоматически подсвечивается. **Жёлтым** через 3 месяца с подписью «давно не обновлялся (3 мес.)». **Красным** через 6 месяцев с подписью «давно не обновлялся (6 мес.), можно убрать». Это подсказка что зрители скорее всего «забыли» про эту запись и её проще удалить чем держать в очереди. Таймер обнуляется когда правишь сумму через UI — БД сама ставит свежий `updated_at`. Сейчас все записи только что засеяны, поэтому подсветки пока не видно — появится со временем.
- **Главная страница — переделана под макет редакторской газеты.** Сверху три **обзорные карточки** последних публикаций с цветным наложением по типу (фиолетово-магента для альбомов, магента-фиолетовая для баттлов, голубо-зелёная для треков) и крупной оценкой в углу. Ниже — **поиск + сортировка** (новые / лучшие / спорные) + **период** (всё время / 30 дней / 7 дней). Под ними — **овальные кнопки-табы** с цветными счётчиками: «Все», «● Альбомы», «⚔ Баттлы», «♪ Треки» (значки для каждого типа). Активный таб — фиолетово-магента градиент. Справа от сетки записей — **закреплённая боковая колонка** «Топ месяца» (5 лучших оценок за 30 дней с цветным скором — жёлтым для 9+, зелёным для 7.5+, голубым для 6+ и т.д.), «Свежие баттлы», «На реакции» (заготовка). Внизу — **полоса активности**: альбомов/баттлов/треков за неделю + всего опубликовано + «Самая высокая оценка месяца». Карточки и оценки везде используют градиенты (как в макете): жёлто-оранжевый для топа, зелёно-голубой для high, голубо-фиолетовый для mid, оранжево-красный для low, красно-бордовый для bad.
- **Шапка сайта — переделана под макет.** Логотип теперь квадрат «R1» с градиентом cyan→purple + жирное «R1fрейтинг» рядом. Навигация: **Каталог · Аукционы · Правила · Админка · Выйти** (последние два — только для админов). Заголовок «Каталог» на главной уменьшен (был слишком большой). Большой счётчик «опубликовано» рядом с заголовком убран.
- **Админские кнопки скрыты от обычных пользователей.** Раньше любой зритель видел в шапке ссылку «Админка» и на главной кнопку «Начать оценку» — это могло сбить с толку. Теперь это видно **только когда залогинен админ**. Зритель видит только каталог и страницы записей с оценками. Доступ в админку остаётся по прямой ссылке `/admin/login` — для админов.
- **Зафиксировали правила сохранности данных.** Перед передачей каталога Рифмабесу написали полный runbook [docs/DATA_SAFETY.md](DATA_SAFETY.md): где живут данные, что их может разрушить, как делать снапшоты, как восстанавливаться. Главное — обновления фронта НЕ трогают данные. Папка `db/snapshots/` для локальных JSON-снапшотов теперь существует (gitignored).
- **Селекторы вместо ручного ввода жанра.** В редакторе альбома и трека вместо текстового поля «Жанр» теперь выпадающий список (Рэп, Поп, R&B, Рок, Электроника, Инди, Метал, Экспериментальный) + опция «Свой» для редких случаев. У баттла поле жанра скрыто — он автоматически «Баттл-рэп». Заодно у баттла появилась полноценная классификация: **Формат** (1 на 1 / 2 на 2 / 3 на 3 / Triple Threat / Дезматч / Свой), **Стиль** (A Capella / BPM / BPM + A Capella / Фристайл / Тематический / Свой), **Стадия** (текст — 1/4, Финал, Fresh Blood, Титульный). YouTube-парсер заполняет формат и стадию автоматически. Меньше опечаток, проще каталогизировать и потом фильтровать.
- **Импорт баттлов с YouTube.** В редакторе при типе «баттл» появился блок «Импорт с YouTube» — вставляешь ссылку на ролик (Кубок МЦ, Versus, и т.п.) → автоматически заполняются название, обложка, стороны A и B (если в названии есть `vs`), плюс ссылка на оригинал. Дезматч-формат (БОЛЬШЕ ЧЕМ БАТТЛ, N участников через `x`) распознаётся, но требует ручного дополнения — модель данных пока знает только две стороны.
- **Новая версия сайта (React + Supabase + Яндекс-импорт) задеплоена на прод.** Раньше на `kirillmaller.github.io/album-rater/` была старая одностраничная HTML-версия — просто калькулятор без сохранения. Теперь там полноценный каталог: можно заводить альбомы / треки / баттлы, ставить оценки, публиковать. Данные хранятся в облачной базе.
- **Импорт из Яндекс.Музыки заработал на проде.** Вставляешь ссылку → нажимаешь «Импортировать» → форма заполняется: название, артист, год, жанр, треклист, **обложка**. Работает и на локалке, и на боевом сайте. Проверено в браузере: альбомы и треки сохраняются корректно. Баттлы тоже сохраняются — пока заполняются руками (импорт баттлов с YouTube запланирован отдельной задачей).
- Заведена документация проекта: корневой `CLAUDE.md`, `docs/ARCHITECTURE.md`, `docs/BACKLOG.md`, `docs/CHANGELOG.md`, спринт в `team/sprints/`. Зафиксировано продуктовое видение: сайт для стримера Рифмабеса, в перспективе — лёгкий логин для зрителей и две независимые оценки (стримера и зрителей).
