# CHANGELOG — R1fрейтинг

Что изменилось для пользователя. Без технического жаргона.

Формат: новые записи сверху, дата в формате `YYYY-MM-DD`.

---

## 2026-05-18

- **Принудительный HTTPS включён.** Сертификат Let's Encrypt выпущен GitHub'ом для `rifmabes.ru`. Теперь сайт открывается только через HTTPS — если кто-то введёт `http://`, его автоматически перебросит на `https://` с замочком 🔒. Пароли при логине шифруются.
- **Иерархия для баттлов: площадка → сезон.** В метаданных баттла появились поля «Площадка» (Versus, SLOVO, Кубок МЦ, 140 BPM, …) и «Сезон» (Fresh Blood 4 и т.п.). При клике на таб «Баттлы» на главной появляется второй ряд табов с площадками; при выборе площадки — третий ряд с её сезонами. Удобно ориентироваться когда баттлов станет больше. В редакторе баттла оба поля с автокомплитом из уже занесённых записей.
- **В каталог заведено 26 черновиков FB4** (Versus: Fresh Blood 4, 2018). Пока они не опубликованы — видны только админу. Когда Кирилл проставит победителей раундов и нажмёт «Опубликовать» — карточки появятся для зрителей.
- **«11 из 10» — пасхалка.** Верхняя граница оценки теперь 11, а не 10. Если трек или альбом реально снёс крышу — можно поставить ему 11, и это сохранится. Spinal Tap-стайл, «this one goes to eleven». Ограничение на стороне БД тоже подняли до 11.
- **Дашборд переделан, чтобы каждый блок имел смысл.** Сверху три **одинаковые** featured-карточки: лучший альбом, лучший баттл, лучший трек **за всё время** (вместо «3 последних», которые дублировали каталог снизу). В правой колонке — **три блока «Топ-3 за выбранный период»** для альбомов / баттлов / треков (период берётся из главного селектора). Убраны блоки «Свежие баттлы» и «На реакции» — первый дублировал каталог, второй был бесполезен. В activity-strip внизу убрана строка «Самая высокая оценка месяца» (тоже дубль). Никаких повторов на экране, каждый блок отвечает на свой вопрос: сверху «лучшее за всё время», в центре «весь каталог», справа «лучшее за период», внизу «активность».
- **«Спорные» → «Худшие».** Третья вкладка сортировки переименована — короче и однозначно.
- **Тип-чип трека** теперь без иконки (только текст «ТРЕК»), а у альбома и баттла — иконки SVG из Lucide (диск / мечи) вместо unicode-символов. Раньше нота `♪` рендерилась криво из-за нестандартной метрики в шрифте Inter. Все три чипа теперь одинаковой ширины 72px.
- **Свой домен `rifmabes.ru`.** Теперь сайт открывается по короткому адресу без `kirillmaller.github.io/album-rater` в URL. Технически по-прежнему хостится на GitHub Pages, но снаружи это незаметно. HTTPS-сертификат GitHub выпустит автоматически через ~30 минут после привязки DNS. После этого можно будет включить принудительный HTTPS.
- **Сайт адаптируется под все размеры экранов.** Десктоп, ноутбук, планшет, телефон — везде раскладка теперь нормальная. Таблица аукционов на узких экранах скроллится горизонтально вместо переноса по строкам. На телефоне шапка превращается в колонку, кнопки в админке растягиваются на ширину экрана, бренд-логотип чуть мельче.
- **Админка приведена в порядок под стиль главной.** Заголовок «Админка» уменьшен (был слишком большой). Табы «Альбомы / Баттлы / Треки» — теперь овальные pill-кнопки с фиолетово-магента градиентом для активного, как на главной. Счётчики записей — в круглых капсулах.
- **Дата выхода + дата оценки.** В редакторе альбома, баттла и трека есть поле «Дата оценки». При создании новой записи автоматически проставляется сегодня (Москва). Если знаешь точную дату когда оценил — можно поставить руками задним числом. На карточках и странице записи теперь видно обе вещи: когда релиз вышел (`вышел: 2024`) и когда был оценён (`оценён: 12 мая 2026`).
- **Поля оценок: точка ИЛИ запятая, без скролла мышью.** Раньше когда листал страницу с открытым полем оценки — колёсико случайно меняло значение. Теперь колёсико снимает фокус, цифра не сбивается. Можно вводить и `5.5`, и `5,5` — оба варианта сохраняются как 5.5. Стрелочки-спинеры справа от поля убраны.
- **Все даты на сайте — в московском времени.** Если показываем дату или время сохранения чего-либо — теперь однозначно МСК. Зрители в любом часовом поясе видят одно и то же.
- **Подробные сообщения об ошибках.** Если сохранение записи падает — теперь в красной плашке показывается реальная причина (например, «дубликат slug», «нет прав на запись»), а не общее «Не удалось сохранить запись». Полная инфа об ошибке также пишется в консоль браузера для отладки.
- **Маленькая кнопка сброса черновика → совсем убрана.** Сначала большая кнопка «Сбросить локальный черновик» уменьшилась до иконки-корзинки, потом вся плашка «Локальный черновик автосохранён в этом браузере» убрана с глаз. Автосохранение работает в фоне без UI-уведомлений.
- **Аукционы — отдельный раздел сайта.** В шапке появились две новые ссылки: **«Аукционы»** и **«Правила»**. Аукционы — это очередь записей, на которые зрители скидывают донаты: чем больше собрано на запись — тем раньше Рифмабес её разбирает. Шесть категорий: альбомы, сериалы, фильмы, аниме, игры, баттлы. Переключаются вкладками, чтобы не листать длинный список. По умолчанию открываются «Альбомы». В очередь сразу залиты 41 запись со старых табличек: 26 альбомов, 10 аниме, 5 сериалов. Категории «фильмы», «игры» и «баттлы» пока пустые. Несколько записей (Киберпанк, Необъятный океан) помечены «Категория под вопросом — уточнить», поправь через админку. **Админ видит дополнительно**: кнопку «+ Добавить» над таблицей и иконки «Изменить» / «Удалить» у каждой строки прямо на публичной странице — не нужно прыгать в админку. Также есть отдельная страница `/admin/auctions` с тем же функционалом, плюс редактор правил.
- **Подсветка «зависших» аукционов.** Если запись с маленькой суммой (≤ 500 ₽) долго не обновляется (никто не докидывает донаты) — она автоматически подсвечивается. **Жёлтым** через 3 месяца с подписью «давно не обновлялся (3 мес.)». **Красным** через 6 месяцев с подписью «давно не обновлялся (6 мес.), можно убрать». Это подсказка что зрители скорее всего «забыли» про эту запись и её проще удалить чем держать в очереди. Таймер обнуляется когда правишь сумму через UI — БД сама ставит свежий `updated_at`. Сейчас все записи только что засеяны, поэтому подсветки пока не видно — появится со временем.
- **Главная страница — переделана под макет редакторской газеты.** Сверху три **лучшие карточки по типам**: лучший альбом, лучший баттл и лучший трек в выбранном периоде. Обложка занимает всю карточку, текст лежит поверх неё, а бейдж типа («альбом», «баттл», «трек») стоит в правом нижнем углу обложки и не съедает место у названия. Ниже — **поиск + сортировка** (новые / лучшие / худшие), **период** (за всё время / за месяц / за 7 дней / в этом году / за прошлый год) и переключатель **оценённые / вышедшие**. Справа от сетки — боковая колонка с лучшими альбомами, баттлами и треками по той же логике периода. Карточки стали чуть светлее, а фон сайта — мягче.
- **Шапка сайта — переделана под макет.** Логотип теперь квадрат «R1» с градиентом cyan→purple + жирное «R1fрейтинг» рядом. Навигация: **Каталог · Аукционы · Правила · Админка · Выйти** (последние два — только для админов). Заголовок «Каталог» на главной уменьшен (был слишком большой). Большой счётчик «опубликовано» рядом с заголовком убран.
- **Админские кнопки скрыты от обычных пользователей, но доступны быстро админу.** Раньше любой зритель видел в шапке ссылку «Админка» и на главной кнопку «Начать оценку» — это могло сбить с толку. Теперь это видно **только когда залогинен админ**. Админ при этом может нажать «Править» прямо на карточке каталога, большой карточке или странице записи, без поиска нужного элемента в списке.
- **Зафиксировали правила сохранности данных.** Перед передачей каталога Рифмабесу написали полный runbook [docs/DATA_SAFETY.md](DATA_SAFETY.md): где живут данные, что их может разрушить, как делать снапшоты, как восстанавливаться. Главное — обновления фронта НЕ трогают данные. Папка `db/snapshots/` для локальных JSON-снапшотов теперь существует (gitignored).
- **Селекторы вместо ручного ввода жанра.** В редакторе альбома и трека вместо текстового поля «Жанр» теперь выпадающий список (Рэп, Поп, R&B, Рок, Электроника, Инди, Метал, Экспериментальный) + опция «Свой» для редких случаев. У баттла поле жанра скрыто — он автоматически «Баттл-рэп». Заодно у баттла появилась полноценная классификация: **Формат** (1 на 1 / 2 на 2 / 3 на 3 / Triple Threat / Дезматч / Свой), **Стиль** (A Capella / BPM / BPM + A Capella / Фристайл / Тематический / Свой), **Стадия** (текст — 1/4, Финал, Fresh Blood, Титульный). YouTube-парсер заполняет формат и стадию автоматически. Меньше опечаток, проще каталогизировать и потом фильтровать.
- **Импорт баттлов с YouTube.** В редакторе при типе «баттл» появился блок «Импорт с YouTube» — вставляешь ссылку на ролик (Кубок МЦ, Versus, и т.п.) → автоматически заполняются название, обложка, стороны A и B (если в названии есть `vs`), плюс ссылка на оригинал. Дезматч-формат (БОЛЬШЕ ЧЕМ БАТТЛ, N участников через `x`) распознаётся, но требует ручного дополнения — модель данных пока знает только две стороны.
- **Новая версия сайта (React + Supabase + Яндекс-импорт) задеплоена на прод.** Раньше на `kirillmaller.github.io/album-rater/` была старая одностраничная HTML-версия — просто калькулятор без сохранения. Теперь там полноценный каталог: можно заводить альбомы / треки / баттлы, ставить оценки, публиковать. Данные хранятся в облачной базе.
- **Импорт из Яндекс.Музыки заработал на проде.** Вставляешь ссылку → нажимаешь «Импортировать» → форма заполняется: название, артист, год, жанр, треклист, **обложка**. Работает и на локалке, и на боевом сайте. Проверено в браузере: альбомы и треки сохраняются корректно. Баттлы тоже сохраняются — пока заполняются руками (импорт баттлов с YouTube запланирован отдельной задачей).
- Заведена документация проекта: корневой `CLAUDE.md`, `docs/ARCHITECTURE.md`, `docs/BACKLOG.md`, `docs/CHANGELOG.md`, спринт в `team/sprints/`. Зафиксировано продуктовое видение: сайт для стримера Рифмабеса, в перспективе — лёгкий логин для зрителей и две независимые оценки (стримера и зрителей).
